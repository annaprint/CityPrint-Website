# CLAUDE.md -- Сайт типографии "Сити Принт"

## Контекст проекта

Новый сайт типографии "Сити Принт" на Next.js (SSG) + Tailwind CSS + TypeScript.
Цель -- конверсия посетителей из рекламы в заявки.

Подробности:
- Бриф проекта (аудитория, цели, тон, стиль): `docs/brief.md`
- Структура сайта и страницы: `docs/sitemap.md`
- Техническая спецификация: `docs/superpowers/specs/2026-04-19-citi-print-website-design.md`

## Правила работы в этом проекте

### Приоритеты при любом решении

1. Понятность для клиента
2. Конверсия в заявку
3. Доверие к типографии
4. Скорость загрузки
5. Визуальная чистота
6. Техническая изящность

Если красивое решение мешает пониманию или конверсии -- отказываемся от него.

### Дизайн

- Много белого пространства. Элементы "дышат", ничего не теснится.
- Фирменный красный (#c41e2a) -- только точечно: CTA-кнопки, ключевые цифры, лейблы.
- Основной фон: теплый белый (#fafaf8). Текст: темный (#1a1a1a).
- Типографика решает больше, чем декор. Контрастные размеры, тонкие начертания заголовков.
- Никаких тяжелых анимаций, параллаксов, декоративных эффектов ради красоты.
- Продукция показывается крупными качественными фото, не стоковыми картинками.
- Дизайн-референсы: imprimeriedumarais.com, kingsburypress.com, moo.com -- см. docs/brief.md.

### Тексты

- Конкретно, уверенно, без канцелярита и штампов.
- Не "комплекс полиграфических решений", а "печатаем книги, каталоги, упаковку".
- Каждый текст отвечает на вопрос клиента: что делаете, сколько стоит, почему вам доверять, что делать дальше.
- Не писать SEO-тексты ради SEO. Страницы должны быть полезными.

### Формы

- Минимум полей: имя, телефон/email, тип продукции, тираж, комментарий, файл.
- Понятные подписи, состояния ошибок, подтверждение отправки.
- Данные отправляются в Битрикс24 REST API.

### Код

- Компоненты переиспользуемые, адаптивность сразу.
- Mobile-first. Кнопка звонка и Telegram фиксированы внизу на мобильных.
- Оптимизация изображений через next/image (WebP, lazy loading).
- Lighthouse 90+ по всем метрикам.

### Чего не делать

- Не добавлять модные эффекты без пользы для конверсии.
- Не использовать стоковые фото как основу визуала.
- Не писать абстрактные офферы ("индивидуальный подход", "высокое качество") без конкретики.
- Не усложнять интерфейс там, где можно упростить.
- Не жертвовать скоростью ради декоративности.
- Не дублировать страницы по смыслу.

### Если данных не хватает

- Не писать абстрактную ерунду.
- Зафиксировать, каких данных не хватает.
- Предложить рабочую гипотезу и продолжить.
