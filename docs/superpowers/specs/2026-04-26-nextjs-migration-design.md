# Спецификация: Перенос сайта Сити Принт в Next.js

## Цель

Перенести прототип сайта (handoff 3) в полноценный Next.js-проект с интеграцией базы знаний, SEO, соответствием 152-ФЗ и деплоем на VDS.

## Контекст

- Прототип: один HTML-файл + JSX-компоненты через Babel (handoff 3)
- 9 продуктовых лендингов, главная, портфолио, о компании
- 26 статей базы знаний в markdown (5 разделов)
- Дизайн: Atelier + Presse, Fraunces + Manrope, тёплый белый фон
- Целевая аудитория: B2B, корпоративные заказчики
- Хостинг: VDS на Net Angels (Node.js, серверы в РФ)
- Репозиторий: github.com/annaprint/CityPrint-Website
- Превью: Vercel (на время разработки)

## Архитектура

### Стек

- **Next.js 14** (App Router)
- **TypeScript**
- **Tailwind CSS** (стили из прототипа переносятся в Tailwind-утилиты + CSS-модули для сложных компонентов)
- **MDX** для статей базы знаний
- **next-sitemap** для SEO
- **PM2 + Nginx** на VDS


## Данные и контент

### Продуктовые тексты
Источник: `тексты-лендингов/*.md` → `src/data/landing-data.ts`
Уже перенесены в landing-data.jsx, конвертируем в TypeScript.

### База знаний
Источник: `content/baza-znaniy/**/*.md` → `src/content/knowledge/**/*.mdx`
26 статей в 5 разделах:
1. Бумага и материалы (5 статей)
2. Подготовка макета (6 статей)
3. Цвет в печати (4 статьи)
4. Технологии и отделка (6 статей)
5. Заказ и экономика (5 статей)

Каждая статья содержит frontmatter (title, description, category, order) и рендерится через MDX.

### SEO
- `title` и `description` для каждого лендинга (уже в landing-data)
- `title` и `description` для каждой статьи (из frontmatter)
- Автоматический sitemap.xml через next-sitemap
- robots.txt
- Микроразметка: FAQPage (лендинги), Article (статьи), LocalBusiness (главная), BreadcrumbList (все страницы)

## Формы

Все формы отправляют данные в **Битрикс24 REST API**.

Поля:
- Имя (обязательно)
- Телефон (обязательно)
- Тип продукции / комментарий
- Файл (опционально)
- Согласие на обработку ПД (чекбокс, обязательно, не отмечен по умолчанию)

Кнопка отправки неактивна без заполненных обязательных полей и чекбокса.

## 152-ФЗ

1. **Политика конфиденциальности** — страница `/privacy`, ссылка в футере
2. **Чекбокс согласия** — в каждой форме, не отмечен по умолчанию
3. **Cookie-баннер** — при первом посещении, выбор сохраняется в localStorage
4. **Сервер в РФ** — VDS у Net Angels

## CMS для редактирования контента

Для редактирования текстов без программиста — **Tina CMS** (headless, open source, визуальный редактор).

Что это даёт:
- Редактирование текстов лендингов, статей базы знаний, страниц прямо в браузере
- Визуальный редактор поверх сайта (видишь страницу и правишь текст на месте)
- Изменения сохраняются в GitHub → автодеплой
- Бесплатный тариф для одного пользователя
- Не нужен отдельный сервер для CMS

Альтернатива: Sanity, Strapi — но Tina проще всего встраивается в Next.js и работает с markdown/MDX.

## Интеграции (делает программист, мы готовим точки подключения)

### Личный кабинет клиента
- Разрабатывается отдельно программистом
- На сайте предусматриваем: кнопку "Личный кабинет" в хедере, страницу-заглушку `/account`
- Когда кабинет будет готов — встраивается через iframe или API

### Онлайн-оплата
- Привязывается программистом к личному кабинету
- На сайте предусматриваем место в интерфейсе

### Калькулятор (Print Effect)
- Полноценный калькулятор разрабатывается программистом на Print Effect
- Временно: калькуляторы из Excel-таблиц (данные от Анны)
- Когда Print Effect будет готов — заменяем блок калькулятора на интеграцию

## Что НЕ входит в текущую разработку

- Интернет-магазин с корзиной
- Рассылка (можно добавить позже)
- Сам личный кабинет и оплата (делает программист)
